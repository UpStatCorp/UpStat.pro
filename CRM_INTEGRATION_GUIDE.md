# 🔌 Руководство по интеграции CRM систем

Эта система позволяет интегрировать вашу CRM для автоматического импорта и анализа записей звонков.

## 📋 Поддерживаемые CRM системы

- ✅ **AmoCRM** - полная поддержка
- ✅ **Bitrix24** - полная поддержка
- 🔜 Salesforce - в разработке

## 🚀 Быстрый старт

### 1. Настройка AmoCRM

1. **Создание интеграции в AmoCRM:**
   - Перейдите в AmoCRM → Настройки → Интеграции → Создать интеграцию
   - Выберите тип "Внешняя интеграция"
   - Укажите название: "UpStat Integration"
   - Redirect URI: `http://localhost:8000/crm/oauth/callback` (для локальной разработки)
   - Для продакшена: `https://вашдомен.com/crm/oauth/callback`
   - Сохраните Client ID и Client Secret

2. **Настройка переменных окружения:**
   ```bash
   # Скопируйте файл примера
   cp env.example .env
   
   # Откройте .env и добавьте:
   AMOCRM_CLIENT_ID=ваш_client_id
   AMOCRM_CLIENT_SECRET=ваш_client_secret
   AMOCRM_REDIRECT_URI=http://localhost:8000/crm/oauth/callback
   ```

### 1.1. Настройка Bitrix24

Bitrix24 поддерживает **два способа подключения**:

#### Вариант 1: Входящий вебхук (рекомендуется для начала) ⭐

**Преимущества:**
- ✅ Быстрая настройка (5 минут)
- ✅ Не требует создания приложения
- ✅ Не требует настройки .env файлов
- ✅ Подходит для личного использования

**Как подключить:**

1. **Откройте ваш портал Bitrix24**
2. **Перейдите:** Приложения → Вебхуки
3. **Нажмите:** "Добавить входящий вебхук"
4. **Настройте права доступа:**
   - ✅ **crm** - доступ к CRM данным
   - ✅ **telephony** - доступ к записям звонков
   - ✅ **user** - информация о пользователях
5. **Сохраните** и скопируйте полный URL вебхука
   - Формат: `https://ваш-портал.bitrix24.ru/rest/1/abc123.../`
6. **В нашей системе:**
   - Откройте "CRM Интеграции"
   - Нажмите "Подключить" на карточке Bitrix24
   - Выберите "Входящий вебхук"
   - Вставьте скопированный URL
   - Нажмите "Подключить"

#### Вариант 2: OAuth приложение (для продакшена)

**Преимущества:**
- ✅ Безопаснее для корпоративного использования
- ✅ Можно использовать на нескольких порталах
- ✅ Автоматическое обновление токенов

**Как подключить:**

1. **Создание приложения в Bitrix24:**
   - Перейдите на https://www.bitrix24.ru/apps/ 
   - Или в вашем портале: Приложения → Разработчикам → Другое → Приложения
   - Нажмите "Создать приложение"
   - Тип: "Серверное приложение"
   - Название: "UpStat Integration"
   - URL обработчика: `http://localhost:8000/crm/oauth/callback` (для локальной разработки)
   - Для продакшена: `https://вашдомен.com/crm/oauth/callback`
   - Права доступа:
     - `crm` - доступ к CRM
     - `telephony` - доступ к телефонии и записям
     - `user` - информация о пользователях
   - Сохраните Client ID и Client Secret

2. **Настройка переменных окружения:**
   ```bash
   # Откройте .env и добавьте:
   BITRIX24_CLIENT_ID=ваш_client_id
   BITRIX24_CLIENT_SECRET=ваш_client_secret
   BITRIX24_REDIRECT_URI=http://localhost:8000/crm/oauth/callback
   ```

3. **В нашей системе:**
   - Откройте "CRM Интеграции"
   - Нажмите "Подключить" на карточке Bitrix24
   - Выберите "OAuth приложение"
   - Введите домен портала
   - Авторизуйтесь в Bitrix24

3. **Применение миграций базы данных:**
   ```bash
   # SQLite (для разработки)
   python -c "from app.database import engine; from app.models import Base; Base.metadata.create_all(bind=engine)"
   
   # Или через Alembic
   alembic upgrade head
   ```

### 2. Подключение CRM в интерфейсе

1. Зайдите в систему
2. Перейдите в раздел **"CRM Интеграции"** в боковом меню
3. Нажмите **"Подключить"** на карточке AmoCRM
4. Введите домен вашего аккаунта (например, `example` для `example.amocrm.ru`)
5. Авторизуйтесь в AmoCRM
6. Система автоматически получит доступ к записям

### 3. Работа с записями

После подключения CRM:

1. **Синхронизация записей:**
   - Нажмите кнопку "Синхронизировать" на карточке интеграции
   - Система получит список записей за последние 30 дней

2. **Анализ записей:**
   - Перейдите в "Записи из CRM" 
   - Выберите нужные записи
   - Нажмите "Анализировать" для каждой записи

3. **Просмотр результатов:**
   - После анализа записи появятся в "Истории звонков"
   - Можно создавать планы тренировок как для обычных записей

## 🛠️ Техническая архитектура

### Структура данных

```python
# CRMIntegration - настройки интеграции
class CRMIntegration:
    - user_id           # Владелец интеграции
    - crm_type          # Тип CRM (amocrm, bitrix24, etc)
    - access_token      # Зашифрованный токен доступа
    - refresh_token     # Зашифрованный refresh токен
    - last_sync_at      # Последняя синхронизация

# CRMRecording - записи из CRM
class CRMRecording:
    - crm_record_id     # ID записи в CRM
    - call_date         # Дата звонка
    - manager_name      # Имя менеджера
    - client_name       # Имя клиента
    - recording_url     # URL записи в CRM
    - sync_status       # Статус обработки
    - conversation_id   # Связь с анализом
```

### Процесс обработки

1. **Получение списка записей** → API CRM
2. **Сохранение метаданных** → База данных
3. **Скачивание по запросу** → При нажатии "Анализировать"
4. **Создание Conversation** → Как для обычной загрузки
5. **Запуск pipeline** → Транскрипция и анализ
6. **Сохранение результатов** → История звонков

### API Endpoints

```http
# Страница интеграций
GET /crm

# Подключение CRM
POST /crm/connect/{crm_type}

# OAuth callback
GET /crm/oauth/callback

# Синхронизация записей
POST /crm/sync/{integration_id}

# Список записей
GET /crm/recordings

# Анализ записи
POST /crm/recordings/{recording_id}/analyze

# Удаление интеграции
DELETE /crm/integrations/{integration_id}
```

## 🔐 Безопасность

1. **Шифрование токенов:**
   - Все OAuth токены шифруются перед сохранением
   - Используется Fernet (симметричное шифрование)
   - Ключ генерируется автоматически или задается в `CRM_ENCRYPTION_KEY`

2. **Проверка прав доступа:**
   - Пользователь видит только свои интеграции
   - Проверка user_id на всех операциях

3. **OAuth безопасность:**
   - State параметр для защиты от CSRF
   - Автоматическое обновление токенов

## 🚨 Решение проблем

### "Ключ AmoCRM не принят"
- Проверьте правильность CLIENT_ID и CLIENT_SECRET
- Убедитесь, что Redirect URI совпадает с настройками в AmoCRM

### "Ошибка подключения Bitrix24"

**Для вебхука:**
- Проверьте, что скопировали полный URL вебхука (включая домен и путь)
- Убедитесь, что вебхук активен в настройках Bitrix24
- Проверьте, что заданы все необходимые права (crm, telephony, user)
- URL должен иметь формат: `https://portal.bitrix24.ru/rest/1/код.../`

**Для OAuth:**
- Проверьте правильность CLIENT_ID и CLIENT_SECRET
- Убедитесь, что Redirect URI совпадает с настройками в приложении
- Проверьте, что приложение активно и одобрено
- Убедитесь, что указаны все необходимые права доступа (crm, telephony, user)

### "Не удалось скачать файл"
- Проверьте права доступа интеграции в CRM
- Убедитесь, что токен не истек (попробуйте переподключить)
- Для Bitrix24: проверьте, что у вас подключена телефония Voximplant

### "Ошибка анализа"
- Проверьте формат аудио (поддерживаются mp3, wav, m4a, opus)
- Убедитесь, что файл не поврежден
