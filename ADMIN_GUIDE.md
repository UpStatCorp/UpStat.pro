# Руководство по админ-панели

## Обзор

В приложение добавлена система ролей пользователей с админ-панелью для управления пользователями.

## Роли пользователей

- **user** - обычный пользователь (по умолчанию)
- **admin** - администратор с доступом к админ-панели

## Создание первого администратора

Для создания первого администратора используйте скрипт:

```bash
# Через Docker
docker-compose run --rm backend python create_admin.py admin@example.com your_password

# Или локально (если Python установлен)
cd app
python create_admin.py admin@example.com your_password
```

## Доступ к админ-панели

1. Войдите в систему с учетными данными администратора
2. В боковом меню появится пункт "Админ-панель" (только для администраторов)
3. Перейдите по ссылке `/admin` для доступа к админ-панели

## Функции админ-панели

### Главная страница (`/admin`)
- Статистика системы (количество пользователей, диалогов, встреч)
- Быстрый доступ к основным функциям

### Управление пользователями (`/admin/users`)
- Просмотр всех пользователей
- Назначение/снятие прав администратора
- Удаление пользователей
- Просмотр информации о пользователях

## Безопасность

- Только пользователи с ролью `admin` могут получить доступ к админ-панели
- Администратор не может изменить собственную роль
- Администратор не может удалить самого себя
- Все действия в админ-панели требуют подтверждения

## API для разработчиков

### Декораторы

```python
from admin import admin_required, get_current_user, is_admin

@admin_required
def admin_only_function(request, db):
    # Эта функция доступна только администраторам
    pass

def some_function(request, db):
    user = get_current_user(request, db)
    if is_admin(user):
        # Логика для администраторов
        pass
```

### Проверка ролей в шаблонах

```html
{% if user and user.role == 'admin' %}
    <!-- Контент только для администраторов -->
{% endif %}
```

## Миграции базы данных

Система использует Alembic для управления миграциями:

```bash
# Запуск миграций через Docker
docker-compose run --rm backend python -m alembic upgrade head

# Создание новой миграции
docker-compose run --rm backend python -m alembic revision --autogenerate -m "Описание изменений"
```

## Устранение неполадок

### Проблема: "Недостаточно прав для доступа к админ-панели"
- Убедитесь, что пользователь имеет роль `admin`
- Проверьте, что поле `role` добавлено в базу данных

### Проблема: Миграции не применяются
- Убедитесь, что файлы alembic находятся в папке `app/`
- Проверьте конфигурацию в `alembic.ini`

### Проблема: Админ-панель не отображается в меню
- Убедитесь, что в шаблоне `_layout_dashboard.html` добавлена проверка роли
- Проверьте, что переменная `user` передается в шаблон
