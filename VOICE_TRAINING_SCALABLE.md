## 🔑 Ключевые особенности

### 1. Изоляция пользователей

Каждый пользователь получает **собственный экземпляр** компонентов:
- **VAD** (Voice Activity Detection) - определение начала/конца речи
- **STT** (Speech-to-Text) - распознавание речи
- **GPT** (Dialogue) - генерация ответов с отдельной историей
- **TTS** (Text-to-Speech) - синтез речи

**Это значит:**
- ✅ История диалога не смешивается между пользователями
- ✅ Один пользователь не блокирует других
- ✅ Ошибка у одного пользователя не влияет на остальных

### 2. Ограничение ресурсов

```python
SessionManager(
    max_concurrent_sessions=100,  # Максимум одновременных сессий
    max_workers=10                # Пул воркеров для STT
)
```

- **Лимит сессий**: При достижении 100 сессий новые подключения отклоняются
- **Пул воркеров**: 10 потоков для блокирующих STT операций
- **Автоочистка**: Неактивные сессии (>1 часа) автоматически закрываются

### 3. Сохранение в БД

Все данные автоматически сохраняются:
- ✅ Создание сессии при подключении
- ✅ Каждое сообщение пользователя и ИИ
- ✅ История диалога с GPT
- ✅ Статистика (длительность, количество сообщений)
- ✅ Завершение или прерывание сессии

### 4. Аутентификация

WebSocket требует JWT токен:
```javascript
wss://server/voice-training/ws?token=<JWT>&training_id=<ID>
```

## 🎯 Использование

### Клиентская сторона

Подключение к WebSocket:

```javascript
// Получаем токен
const token = localStorage.getItem('access_token');

// ID тренировки
const trainingId = 1;

// Подключаемся
const ws = new WebSocket(
  `wss://server/voice-training/ws?token=${token}&training_id=${trainingId}`
);

ws.onopen = () => {
  console.log('✅ Подключено');
};

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  
  switch(data.type) {
    case 'connected':
      console.log('Сессия:', data.session_id);
      break;
    case 'user_text':
      console.log('Вы сказали:', data.text);
      break;
    case 'ai_text':
      console.log('ИИ ответил:', data.text);
      break;
    case 'audio_chunk':
      // Воспроизвести аудио
      playAudio(data.audio);
      break;
  }
};

// Отправка аудио
function sendAudio(audioFloat32Array) {
  const base64 = arrayToBase64(audioFloat32Array);
  ws.send(JSON.stringify({
    type: 'audio',
    audio: base64
  }));
}
```

### Серверная сторона

#### Создать сессию

## ⚡ Производительность

### Текущие ограничения

| Параметр | Значение | Настройка |
|----------|----------|-----------|
| Макс. одновременных сессий | 100 | `SessionManager.max_concurrent_sessions` |
| Воркеры STT | 10 | `SessionManager.max_workers` |
| Таймаут неактивности | 3600 сек | `cleanup_inactive_sessions(timeout_seconds)` |
| Порог тишины для VAD | 0.65 сек | `SILENCE_THRESHOLD` |
| Порог речи (RMS) | 0.02 | `SPEECH_THRESHOLD` |

### Рекомендации по масштабированию

#### Для 100-200 пользователей (текущая конфигурация)
- ✅ Работает на одном сервере
- CPU: 4-8 ядер
- RAM: 8-16 GB
- Не требует изменений

#### Для 200-500 пользователей
1. Увеличить `max_concurrent_sessions` до 500
2. Увеличить `max_workers` до 20-30
3. Использовать более мощный сервер (8-16 ядер, 32 GB RAM)
4. Рассмотреть PostgreSQL вместо SQLite

#### Для 500+ пользователей
1. **Вертикальное масштабирование**:
   - Выделенный сервер для БД (PostgreSQL)
   - Отдельный сервер для FastAPI
   - Redis для хранения состояний сессий

2. **Горизонтальное масштабирование**:
   - Несколько экземпляров FastAPI за Load Balancer
   - Sticky sessions для WebSocket (по user_id)
   - Shared storage для аудио файлов

3. **Микросервисная архитектура**:
   ```
   Load Balancer (nginx/HAProxy)
        │
        ├─── FastAPI Instance 1 (WebSocket)
        ├─── FastAPI Instance 2 (WebSocket)
        └─── FastAPI Instance N (WebSocket)
                │
                ├─── STT Service (отдельный сервис)
                ├─── GPT Service (отдельный сервис)
                ├─── TTS Service (отдельный сервис)
                └─── PostgreSQL + Redis
   ```

## 🐛 Отладка

### Проблема: "Достигнут лимит пользователей"

**Причина**: Все 100 слотов заняты

**Решение**:
1. Проверить статистику: `GET /voice-training/stats`
2. Запустить очистку: `SessionManager.cleanup_inactive_sessions()`
3. Увеличить `max_concurrent_sessions`

### Проблема: "Unauthorized: Invalid token"

**Причина**: Невалидный JWT токен

**Решение**:
1. Проверить что токен передан: `?token=...`
2. Проверить срок действия токена
3. Убедиться что токен правильный (из `/api/login`)

### Проблема: Медленная обработка

**Причина**: STT воркеры заняты

**Решение**:
1. Увеличить `max_workers` в SessionManager
2. Использовать более быстрый STT (например, Whisper tiny вместо base)
3. Добавить кэширование часто используемых фраз

## 📝 API Reference

### WebSocket Messages

#### Клиент → Сервер

| Type | Payload | Описание |
|------|---------|----------|
| `audio` | `{audio: base64}` | Отправка аудио чанка |
| `text` | `{text: string}` | Текстовый запрос (для тестов) |
| `stop` | `{}` | Остановка текущей обработки |
| `end_session` | `{}` | Завершение сессии |

#### Сервер → Клиент

| Type | Payload | Описание |
|------|---------|----------|
| `connected` | `{session_id, message}` | Подключение установлено |
| `status` | `{status, message}` | Изменение статуса |
| `user_text` | `{text}` | Распознанный текст пользователя |
| `ai_text` | `{text}` | Ответ ИИ (текст) |
| `audio_start` | `{message}` | Начало озвучивания |
| `audio_chunk` | `{audio: base64}` | Чанк аудио ответа |
| `audio_end` | `{}` | Конец озвучивания |
| `error` | `{message}` | Ошибка |
| `session_ended` | `{duration, messages_count}` | Сессия завершена |
