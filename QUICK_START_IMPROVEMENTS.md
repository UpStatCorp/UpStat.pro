# 🚀 Быстрый старт: Новые возможности

## 📦 Установка

```bash
# 1. Обновите зависимости
pip install -r requirements.txt

# 2. Пересоберите Docker образы
docker-compose build --no-cache

# 3. Запустите приложение
docker-compose up -d

# 4. Проверьте логи
docker-compose logs -f backend
```

---

## 🎯 Ключевые новые возможности

### 1. 🔔 Система уведомлений

**Где увидеть:** Уведомления появляются в правом верхнем углу экрана

**Что делает:**
- Показывает прогресс обработки файлов
- Уведомляет о завершении анализа
- Сообщает об ошибках с понятными объяснениями

**Пример:**
- Загрузите аудио файл → увидите прогресс-бар
- Дождитесь завершения → получите уведомление "Анализ завершен"

---

### 2. 📊 Мониторинг производительности

**Где найти:** `/admin/performance` (только для администраторов)

**Что можно делать:**
- ✅ Смотреть статистику кеша, БД и хранилища
- ✅ Очищать кеш одной кнопкой
- ✅ Оптимизировать базу данных
- ✅ Удалять старые файлы
- ✅ Видеть загрузку CPU, памяти и диска

**Как использовать:**
1. Войдите как администратор
2. Перейдите на `/admin/performance`
3. Просмотрите статистику
4. При необходимости нажмите кнопку "Оптимизировать"

---

### 3. ⚡ Ускорение работы

**Что изменилось:**
- Запросы к БД работают быстрее (кеширование)
- Повторные запросы одних и тех же данных почти мгновенны
- Файлы обрабатываются в фоне, не блокируя интерфейс

**Как почувствовать:**
- Откройте дашборд → загружается быстрее
- Обновите страницу → данные из кеша
- Загрузите файл → обработка в фоне, можно работать дальше

---

### 4. 🔒 Улучшенная безопасность

**Что добавлено:**
- Проверка реального типа загружаемых файлов
- Ограничение частоты запросов (защита от спама)
- Строгая валидация паролей (минимум 8 символов, цифры и буквы)
- Защита от CSRF и XSS атак

**Как работает:**
- Попробуйте загрузить файл с неправильным расширением → система распознает обман
- Слишком много запросов → временная блокировка
- Слабый пароль при регистрации → система не примет

---

### 5. 📝 Понятные ошибки

**Что изменилось:**
- Вместо технических ошибок → понятные объяснения
- Каждая ошибка содержит совет, что делать

**Примеры:**
- ❌ Старое: "HTTPException 500"
- ✅ Новое: "Не удалось конвертировать аудио файл. Проверьте формат файла и попробуйте еще раз"

## 🔧 API для разработчиков

### Уведомления

```bash
# Получить непрочитанные уведомления
GET /api/notifications/unread

# Отметить как прочитанное
POST /api/notifications/{id}/read

# Очистить все
DELETE /api/notifications/clear-all
```

### Прогресс операций

```bash
# Получить прогресс
GET /api/progress/{operation_id}

# Список активных операций
GET /api/progress/active/list
```

### Производительность (только админы)

```bash
# Статистика кеша
GET /api/performance/cache/stats

# Очистить кеш
POST /api/performance/cache/clear

# Статистика БД
GET /api/performance/database/stats

# Оптимизировать БД
POST /api/performance/database/optimize

# Статистика хранилища
GET /api/performance/storage/stats

# Очистить старые файлы
POST /api/performance/storage/cleanup?days_to_keep=7
```

## 🎨 Использование в коде

### Python (Backend)

```python
# Кеширование
from services.caching_service import cached

@cached('user_profile', ttl=300)
def get_user_profile(user_id: int):
    return expensive_query()

# Уведомления
from services.notification_service import get_notification_service

notif = get_notification_service()
notif.success(user_id, "Успех!", "Файл загружен")
notif.error(user_id, "Ошибка", "Не удалось обработать файл")

# Прогресс
from services.progress_tracker import get_progress_tracker

tracker = get_progress_tracker()
progress = tracker.create_operation("my_op", total_stages=3, title="Обработка...")
tracker.update_operation("my_op", 1, "Шаг 1", "Загрузка...")
tracker.complete_operation("my_op", "Готово!")
```

### JavaScript (Frontend)

```javascript
// Уведомления
window.notifications.success('Успешно сохранено', 'Успех!');
window.notifications.error('Ошибка сохранения', 'Ошибка');
window.notifications.warning('Файл слишком большой', 'Внимание');
window.notifications.info('Новое сообщение', 'Информация');

// Прогресс отслеживается автоматически
// Компонент сам опрашивает сервер каждые 2 секунды
```

## 🐛 Если что-то пошло не так

### Уведомления не показываются
```bash
# Проверьте консоль браузера
# Убедитесь что подключены скрипты:
# - /static/js/notifications.js
# - /static/css/notifications.css
```

### Админ-панель производительности не работает
```bash
# Убедитесь что вы вошли как администратор
# Проверьте что установлен psutil:
pip install psutil
```

### Кеш не работает
```bash
# По умолчанию используется in-memory cache
# Для Redis добавьте в .env:
REDIS_URL=redis://localhost:6379/0
```

## 💡 Советы по использованию

### Для пользователей:
1. Обращайте внимание на уведомления - они помогают понять, что происходит
2. Не закрывайте вкладку во время обработки - увидите прогресс
3. Если видите ошибку - прочитайте сообщение, там есть совет

### Для администраторов:
1. Заходите в `/admin/performance` раз в неделю
2. Очищайте старые файлы и записи БД
3. Следите за размером базы данных
4. При замедлении работы - проверьте загрузку системы

### Для разработчиков:
1. Используйте декоратор `@cached` для частых запросов
2. Добавляйте уведомления при важных действиях
3. Используйте `progress_tracker` для длительных операций
4. Применяйте `bulk_insert` для массовых операций
