# 🚀 Руководство по оптимизации производительности голосового ассистента

## ⚡ Текущие оптимизации (уже применены)

### 1. Уменьшение паузы детекции тишины
- **SILENCE_DURATION_MS**: 650мс (было 1000мс)
- **SILENCE_THRESHOLD**: 0.65 сек (было 1.0 сек)
- **Интервал проверки**: 50мс (было 100мс)

Ассистент теперь начинает обработку **практически мгновенно** после вашей паузы в 650-700мс.

## 🔧 Дополнительные рекомендации для ещё большей скорости

### 2. Выбор оптимального STT провайдера

В файле `.env` установите:

```bash
# Самый быстрый вариант (рекомендуется):
STT_PROVIDER=whisper_openai

# Самый точный, но чуть медленнее:
STT_PROVIDER=elevenlabs

# Локальный (офлайн), но медленнее:
STT_PROVIDER=whisper_local
WHISPER_MODEL=base  # Используйте "base" вместо "small" для скорости
```

**Рекомендация**: `whisper_openai` обеспечивает лучший баланс скорости и точности.

### 3. Оптимизация TTS (Text-to-Speech)

```bash
# Для максимальной скорости используйте OpenAI TTS:
TTS_PROVIDER=openai
TTS_MODEL=tts-1  # НЕ tts-1-hd (HD медленнее)
TTS_VOICE=nova   # или shimmer, alloy

# ElevenLabs качественнее, но медленнее:
TTS_PROVIDER=elevenlabs
ELEVENLABS_MODEL_ID=eleven_turbo_v2  # Turbo версия быстрее
```

### 4. Оптимизация GPT ответов

В файле `.env`:

```bash
GPT_MODEL=gpt-4o-mini  # Быстрее чем gpt-4o
GPT_MAX_TOKENS=100     # Ещё короче для ещё большей скорости (сейчас 150)
GPT_TEMPERATURE=0.7    # Оптимальное значение
```

### 5. Параметры для тонкой настройки

Если хотите **ещё более агрессивную** реакцию (экспериментальные настройки):

```python
# В voice_assistant/config.py можно изменить:
SILENCE_DURATION_MS = 500  # Ещё быстрее! (осторожно - может прерывать речь)
VAD_THRESHOLD = 0.25       # Более чувствительная детекция речи

# В voice_assistant/router.py:
SILENCE_THRESHOLD = 0.5    # 500мс (ещё быстрее, но рискованно)
SPEECH_THRESHOLD = 0.015   # Более низкий порог речи
```

⚠️ **Осторожно**: Слишком низкие значения могут привести к преждевременной обработке (ассистент может "перебивать" вас).

## 📊 Типичная задержка после оптимизации

- **Детекция конца речи**: ~650мс (ваша пауза)
- **STT (Whisper OpenAI)**: 200-500мс
- **GPT генерация**: 300-800мс (зависит от сложности запроса)
- **TTS (OpenAI tts-1)**: 200-400мс
- **Общее время до первого ответа**: **1.4-2.4 секунды**

С ElevenLabs TTS или более сложными запросами время может увеличиться до 2-3 секунд.

## 🎯 Рекомендуемая конфигурация для максимальной скорости

Создайте файл `.env` со следующими параметрами:

```bash
# STT - Speech to Text
STT_PROVIDER=whisper_openai
OPENAI_API_KEY=ваш_ключ_openai

# TTS - Text to Speech
TTS_PROVIDER=openai
TTS_MODEL=tts-1
TTS_VOICE=nova

# GPT
GPT_MODEL=gpt-4o-mini
GPT_MAX_TOKENS=100
GPT_TEMPERATURE=0.7

# Опционально: для ElevenLabs (если предпочитаете качество)
ELEVENLABS_API_KEY=ваш_ключ_elevenlabs
ELEVENLABS_VOICE_ID=ваш_voice_id
ELEVENLABS_MODEL_ID=eleven_turbo_v2
```

## 🧪 Тестирование и отладка

Для тестирования производительности проверьте логи:

```bash
# Логи показывают время обработки каждого этапа
tail -f voice_assistant/logs/assistant.log
```

Ищите строки типа:
- `🎤 Отправка в STT: ...` - начало распознавания
- `🎤 Распознано: ...` - конец распознавания
- `🤖 Генерация ответа...` - GPT обработка
- `🔊 Синтез речи...` - TTS обработка

## 💡 Советы

1. **Говорите чётко**: Чёткая речь = быстрее распознавание
2. **Короткие запросы**: Короткие вопросы = быстрее ответы
3. **Стабильное соединение**: Хороший интернет критически важен для облачных STT/TTS
4. **Используйте проводной микрофон**: Меньше шума = точнее детекция речи
