# Руководство по системе управления промптами

## 🎯 Обзор

Создана полноценная система управления промптами с версионированием, которая позволяет администраторам:
- Создавать и редактировать промпты через веб-интерфейс
- Сохранять историю версий с возможностью отката
- Управлять активными промптами
- Использовать промпты в AI-анализе

## 🚀 Доступ к системе

### Для администраторов:
1. **Войдите в систему** с правами администратора
2. **Перейдите в админ-панель** (кнопка "Админ" в топбаре)
3. **Выберите "Промпты"** в админском меню
4. **Или перейдите напрямую** на `/admin/prompts`

### Учетные данные администратора:
- Email: `admin@example.com`
- Пароль: `admin123`

## 📋 Функции системы

### 1. Просмотр промптов (`/admin/prompts`)
- ✅ Список всех активных промптов
- ✅ Статистика по промптам
- ✅ Быстрые действия (создание, импорт)
- ✅ Информация о версиях и авторах

### 2. Создание промпта (`/admin/prompts/create`)
- ✅ Форма создания нового промпта
- ✅ Системное название (уникальный идентификатор)
- ✅ Человекочитаемое название
- ✅ Описание промпта
- ✅ Содержимое с поддержкой переменных
- ✅ Справка по созданию промптов

### 3. Редактирование промпта (`/admin/prompts/{name}/edit`)
- ✅ Редактирование существующего промпта
- ✅ Создание новой версии при сохранении
- ✅ Автоматическая активация новой версии
- ✅ Информация о текущей версии

### 4. Управление версиями (`/admin/prompts/{name}/versions`)
- ✅ Просмотр всех версий промпта
- ✅ Активация любой версии
- ✅ Удаление неактивных версий
- ✅ История изменений с авторами

## 🔧 Технические особенности

### Версионирование
- **Автоматическое версионирование** при каждом изменении
- **Сохранение истории** всех версий
- **Возможность отката** к любой предыдущей версии
- **Активация версий** с деактивацией предыдущих

### Переменные в промптах
Поддерживаются следующие переменные:
- `{data}` - данные чек-листа
- `{dialogue_json_str}` - JSON диалога
- `{user_name}` - имя пользователя
- `{timestamp}` - текущее время

### Интеграция с pipeline
- **Автоматическое использование** активных промптов
- **Fallback на старый промпт** если в БД нет активного
- **Динамическая подстановка** переменных

## 📊 Структура базы данных

### Таблица `prompts`:
```sql
- id: Primary key
- name: Системное название (sales_audit)
- title: Человекочитаемое название
- description: Описание промпта
- content: Содержимое промпта
- version: Номер версии
- is_active: Активен ли промпт
- created_by: ID создателя
- created_at: Дата создания
- updated_at: Дата обновления
```

## 📝 Примеры использования

### Создание нового промпта:
1. Перейдите в `/admin/prompts/create`
2. Заполните системное название (например, `conversation_analysis`)
3. Укажите отображаемое название
4. Добавьте описание
5. Введите содержимое промпта с переменными
6. Нажмите "Создать промпт"

### Редактирование промпта:
1. Найдите промпт в списке
2. Нажмите "Редактировать"
3. Внесите изменения
4. Нажмите "Сохранить новую версию"
5. Новая версия автоматически станет активной

### Откат к предыдущей версии:
1. Перейдите в "Версии" промпта
2. Найдите нужную версию
3. Нажмите "Активировать"
4. Подтвердите действие

## 🚀 Инициализация

При первом запуске создается базовый промпт `sales_audit` с содержимым из pipeline.


## 🎯 Преимущества системы

1. **Гибкость**: Легко изменять промпты без перезапуска
2. **Безопасность**: История версий с возможностью отката
3. **Удобство**: Веб-интерфейс для управления
4. **Контроль**: Полный контроль над AI-поведением
5. **Масштабируемость**: Легко добавлять новые типы промптов

## 🔄 Workflow

1. **Администратор** создает/редактирует промпт
2. **Система** создает новую версию
3. **Новая версия** автоматически активируется
4. **Pipeline** использует активный промпт
5. **При необходимости** можно откатиться к предыдущей версии

Теперь у вас есть полноценная система управления промптами! 🎉
